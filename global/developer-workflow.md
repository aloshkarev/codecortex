# Developer Workflow

- Prefer plan-first execution for broad, cross-file changes.
- Run impact analysis before refactoring shared modules.
- Keep changes small, testable, and reversible.
- Always run formatting, lints, and tests before finalizing.
- Treat generated config files as managed artifacts.
- Follow `expert-agent-skill-framework.md` for agent/skill structure.
- Follow `naming-conventions.md` when adding or renaming agents/skills.
- Use `review-severity-rubric.md` for review findings.
- Use `debugger-triage-checklist.md` and `optimization-methodology.md` for debugger/optimizer workflows.
